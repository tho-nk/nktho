Architect

The trading exchange has two main components:

Server

This is a server using the UDP protocol, which acts as a listener for client requests and dispatches these requests to a matching engine. The implementation is based on Boost.Asio.

Matching Engine

This component performs the logic of matching order when receiving new client request.

It supports multi orderbook by symbol.

  • Each orderbook maintains order with AtPriceOrderMap. Order with same price will be add to the same AtPriceOrder using time priority. Using hashtable (std::array) support constant lookup. Each AtPriceOrder is linked to the prev and next PriceLevel.

  • Each orderbook maintains Top of Book (TOB) per side.

  • In buy side, the top of the book is the AtPriceOrder with highest buy price. Next AtPriceOrder of the TOB is the next highest smaller AtPriceOrder in the AtPriceOrderMap.

  • In sell side, the top of the book is the AtPriceOrder with lowest sell price. Next AtPriceOrder of the TOB is the next smallest higher AtPriceOrder in the AtPriceOrderMap.

  • Orders are managed with OrderManager, this component is in charged of creating new order and releasing expired order.

FIFO Queue

A thread-safe MPMC (multi-producers, multi-consumers) queue used to communicate between the server and the matching engine

  • Stdout is used to streamed the output of the matching engine. The stdout is used in threadsafe
  • Stdlog is used to streamed the performance measurement message
    • Perf|OrderBook|Add|Execution|111
    • Perf|OrderBook|Cancel|Execution|50

Improvements

  • Add an interface for the Server to support listening on different protocols, such as TCP/IP.
  • Implement the FIX protocol.
  • Implement a lock-free queue.
  • Optimize performance.
  • Implement an object pool to avoid dynamic allocations.
